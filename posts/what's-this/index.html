What’s this?

Technically, A Static Website

This is:

  • A static website.
  • Hosted on Github through Github Pages.
  • Generated by Jekyll.
  • Built based on Jekyll theme jekyll-theme-chirpy.

Functionally, More Than Just A Personal Blog

This is:

  • My personal blog, of course.
  • My personal Media Aggregation Platform or Media Aggregation Portal (MAP). (Cool name, hah, instantly copied & pasted from google :P).
  • My writing playpen.
  • A place for me to show off.

So, how…

How did I build this?

Long story for short,

I tried Wordpress and hosting a dynamic website on Aliyun in the very beginning, finding out it was a little too complicated and has too much overhead for my goal (having a personal website like this).

I happened to learn about Jekyll when I was working on a SDK documentation website for one of my projects, eventually finishing the website with Jekyll in a quick manner. The website received positive feedback and I realized it would be nice to make this website a static website generated by Jekyll.

With the experience gained from the documentation website work, I started to build this website with Jekyll. It took me a while to decide the general style of the site by picking a promising theme. Finally, jekyll-theme-chirpy comes into my life and it was just, beautiful.

Since Jekyll is a static site generator with built-in support for Github Pages and it is also recommended by Github Pages, I thought it would be good to host this website on Github. So, I decided to use Github Pages, and it turned out to be a great and right decision.

Instead of using a Github style domain http(s)://<username>.github.io/<repository>, I chose to use my personal domain name fatdexter.cn.

As a result, this website comes to this world.

And if you want more technical details…

Jekyll and Github Pages both have great documentation so I’m not going to talk about any fundamental details. But, one thing I would like to share is the repos of this website project.

Generall speaking, this website project has 3 different repos:

  1. My customized jekyll-theme-chirpy theme: A private repo forked from the original jekyll-theme-chirpy.

    I use this repo to commit/record my custom changes related to theme layouts into the original theme. For example, the original theme supports dark mode but uses the same font color in both dark mode and light mode. (at least for the version I forked from) However, I would prefer a different font color in the two modes. So, I made a few changes to the css and page files.

    Note that this repo does NOT have any information about the website to publish. It’s just a place for me to tailored the jekyll-theme-chirpy theme for my website.

  2. Jekyll project for the website: A private repo forked from my first repo.

    This is a standard jekyll project based on my customized jekyll-theme-chirpy theme. In other words, I put all the articles, links, strings or anything would be displayed/used in the real website into this repo. And the finally published website will be generated from this repo.

    Note that this repo does NOT have any changes to the theme itself.

  3. Final website files: A public repo contains the generated website from the second repo.

    According to Github Pages’s requirement, GitHub Pages sites and its repo are publicly available on the internet by default and this repo can only be changed to private with GitHub Enterprise Cloud. However, I don’t have GitHub Enterprise Cloud, so I have to create an independent public repo to store the website files and protect my Jekyll project from being made public.

I think that’s all I want to share for now.
